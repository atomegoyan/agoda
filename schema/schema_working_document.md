# Générer le schéma de l'encodage

**Question :** 
- liste fermée, quand les valeurs existent déjà dans la tei (main sub par ex) est-ce qu'il faut quand même créer la liste fermée ?
- est-ce qu'il est possible de restreindre les valeurs différemment sur un même attribut mais pas utilisé dans le même contexte (ex : utilisation de persName dans teiHeader =/ dans text) ?


## 1- Méthodologie pour générer le schéma

**Plusieurs solutions :**
- Roma nouveau
- Roma ancien
- Odd by example

**Méthodes :**
- **Roma ancien** : 
 - étape 1 : choix de la méthode par suppression
 - étape 2 : configuration des paramètres --> titre du fichier... 
 - étape 3 : choix de la langue
 - étape 4 : suppressions des modules inutiles un par un --> cherche pour chaque élément + chaque attribut dans quel module ils sont. Pour vérifier mes choix, comparaison avec le résultat de ODD by example.
 - étape 5 : aller dans chaque module un par un et inclure/ exclure les éléments
 - étape 6 : aller dans chaque élément un par un et inclure/exclure les attributs
 - étape 7 : paramétrer les attributs : optionnel/ obligatoire, liste fermée ou non

- **ODD by example** : génération de l'ODD grâce à Oxygen
